### 🎨 Codex-Identified Color Issues

1. ⚠️ **Violet accent drift in live preview** — Some components (particularly in the live preview and generator UI) use `violet-*` tokens outside the defined `#136dec` blue design system. This creates visual inconsistency. A design system style guide is needed to enforce token usage.
2. ⚠️ **Only one accent color family** — Academic tools should have warmer secondary tones.
3. ⚠️ **No semantic color tokens** — No explicit `success/warning/error/info` colors defined in the design system.
4. ⚠️ **No gradient definitions** in Tailwind config.

**Recommended Token Additions:**
```json
"success": "#22c55e",
"warning": "#f59e0b",
"error": "#ef4444",
"info": "#3b82f6",
"secondary": "#8b5cf6",      /* Purple — for AI/generator features */
"accent": "#14b8a6",         /* Teal — for quality scores */
"surface-1": "#18181b",      /* Dark mode card background */
"surface-2": "#27272a",      /* Dark mode elevated surface */
"text-muted": "#a1a1aa"      /* Muted text */
```

## Critical CSS Issue 🚨

**`globals.css` is 6,337 lines (117KB)** — this is compiled Tailwind output that should **NOT** be committed. It should be auto-generated by PostCSS at build time. This causes:
- Massive initial CSS payload (117KB baseline)
- Duplicate utility classes
- Hard to maintain custom styles
- Fights with Tailwind's JIT compiler

**Fix:** Delete `globals.css` compiled content; keep only custom CSS. Let Tailwind JIT generate utilities automatically.
