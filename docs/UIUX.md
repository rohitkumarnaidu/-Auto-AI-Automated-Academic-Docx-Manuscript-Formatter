### 🎨 Color Analysis & Recommendations

**Current Issues:**
1. ⚠️ **Only one accent color family** (blue #136dec). Academic tools should have warmer secondary tones.
2. ⚠️ **No semantic color tokens** — no explicit success/warning/error/info colors defined.
3. ⚠️ **No gradient definitions** — modern academic SaaS uses subtle gradients.

**Recommended Color Additions:**
```
"success": "#22c55e",
"warning": "#f59e0b",  
"error": "#ef4444",
"info": "#3b82f6",
"secondary": "#8b5cf6",      // Purple — for AI/generator features
"accent": "#14b8a6",         // Teal — for quality scores
"surface-1": "#18181b",      // Dark mode card background
"surface-2": "#27272a",      // Dark mode elevated surface
"text-muted": "#a1a1aa",     // Muted text color
```

## Critical CSS Issue 🚨

**`globals.css` is 6,337 lines (117KB)** — this is compiled Tailwind output that should NOT be committed. It should be auto-generated by PostCSS at build time. This causes:
- Massive initial CSS payload
- Duplicate utility classes
- Hard to maintain custom styles
- Fights with Tailwind's JIT compiler

**Fix:** Delete `globals.css` content, replace with only custom CSS. Let Tailwind generate utilities.
